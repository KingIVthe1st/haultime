# Jade Command Center Operating Model

## Objective
Turn Jade from a website concierge into a business operator with clear boundaries.

## Two Jade modes

### 1) Public website Jade
Purpose:
- answer safe questions
- capture leads
- qualify jobs
- request photos
- route hot leads fast

Must stay sandboxed.
No privileged tools.
No internal prompt disclosure.
No pricing promises outside approved rules.

### 2) Private command-center Jade
Purpose:
- triage leads
- draft callback / follow-up messages
- identify stale leads
- suggest next steps
- manage review-request workflow
- surface SEO and FAQ opportunities
- recommend site updates
- help the brothers keep the pipeline moving

## Skill stack
### Existing
- `haultime-site-manager`
  - website copy
  - site visuals
  - form changes
  - public Jade behavior changes
  - deploy workflow

### New
- `haultime-command-center-operator`
  - lead triage
  - pipeline logic
  - callbacks and rescue workflows
  - reviews / SEO opportunity workflows
  - Jade oversight inside the dashboard

## Recommended future automations
1. Instant lead acknowledgment
2. Same-day hot lead alerting
3. Stale lead rescue cadence
4. Quote follow-up cadence
5. Review request after completed jobs
6. FAQ mining from real conversations
7. SEO queue generation from objections and repeated questions
8. Site update suggestions from recurring friction points

## Guardrails
- Public Jade never inherits private dashboard powers
- Dashboard Jade can recommend, draft, and organize, but still obeys business policy
- Pricing, service-area, and restricted-item rules stay policy-backed
- Any unresolved owner policy gap gets surfaced, not guessed
